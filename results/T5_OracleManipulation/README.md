# T5 DeFi 漏洞合约自动化安全评估报告

## 1. 项目概述

本项目利用 **Echidna**（基于属性的模糊测试工具）对 11 个真实世界的 DeFi 漏洞合约（T5 类别）进行了深度评估。目标合约涵盖了价格预言机操纵、清算逻辑缺陷、底层内存越界等核心安全威胁。

**实验环境：**

- **操作系统:** Linux (Docker 容器: `8446df48161f`)
- **测试工具:** Echidna v2.2.3
- **编译器支持:** 通过 `solc-select` 自动切换多版本（0.5.16 至 0.8.28）

------

## 2. 测试数据集 (T5 漏洞分类)

本次评估涵盖了以下 11 个核心文件，根据其漏洞根源进行了详细分类：

| **漏洞类别**   | **对应文件名**                                               | **核心漏洞点描述**                                           |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **预言机操纵** | `compounduni_vuln.sol`, `uwulend_vuln.sol`, `moonwell_vuln.sol`, `stable_vuln.sol` | 喂价逻辑缺陷、缺少管理员权限校验、稳定币脱锚风险。           |

## 3. 评估方法论

### 自动化批量测试流程

我们开发了集成脚本 `run_all_t5_final.sh` 来实现以下流程：

1. **版本自适应：** 根据不同合约需求，自动切换 `solc` 编译器版本。
2. **测试包装层 (Wrapper)：** 通过继承原合约，自动注入针对性的 **Invariants（不变性属性）**。
3. **压力测试：** 每个合约运行 50,000 次测试序列，尝试触发状态违规。

### 关键性能指标 (KPI)

- **发现时间 (TTF):** 工具找到反例的速度。
- **调用序列长度:** 触发漏洞所需的连续操作步数。
- **代码覆盖率:** 模糊测试触达的合约逻辑百分比。

## 4. 实验结果总结 (Step 4 深度分析)

### 🔴 已检出漏洞 (FAILED)

- **案例:** `compound`, `stable`, `makina`, `pancakeswap`
- **原因分析:** 这些漏洞属于“浅层逻辑错误”或“直接数值风险”。Echidna 的随机模糊算法能极快地在数秒内通过大范围尝试触达错误状态（例如：直接调用未授权的价格更新函数）。

### 🟢 未检出漏洞 (PASSING)

- **案例:** `badger`, `moonwell`, `impermax`
- **原因分析 (Root Cause):** 这反映了自动化工具的局限性。T5 中的复杂 DeFi 逻辑漏洞往往需要**极长的特定操作序列**（如：存款 -> 等待产生利息 -> 触发特定比例清算）。随机模糊测试在没有人工精密引导的情况下，很难通过“盲撞”复现此类高级业务漏洞。
- **结论:** 这证明了在复杂的 DeFi 审计中，工具的效果高度依赖于审计师编写 Invariants 的质量。

## 5. 如何复现

1. 确保当前目录下存在 `config_t5.yaml` 配置文件。

2. 执行批量自动化脚本：

   Bash

   ```
   chmod +x run_all_t5_final.sh
   ./run_all_t5_final.sh
   ```

3. 测试报告将自动生成在 `/share/results/T5_Full_Batch` 目录中。

------

## 6. 结语

本次评估表明，**Echidna** 在捕获“低垂果实”（如内存溢出、简单权限绕过）方面表现卓越。然而，面对 T5 级别的复杂业务逻辑漏洞，自动化工具必须辅以高水平的人工 Invariant 设计，才能发挥其最大价值。
